Электронная доска объявлений (REST API)

О проекте

REST API приложение для электронной доски объявлений, предоставляющее полноценный бэкенд для платформы 
по размещению и управлению объявлениями. 
Система построена на современном Java-стеке с использованием ролевых моделей данных, современных систем защиты и
кеширования.

Проект представляет собой серверное приложение, которое обрабатывает всю бизнес-логику платформы: 
- Размещения и управления объявлениями
- Аутентификации/авторизации пользователей 
- Комментирования объявлений
- Администрирования пользователей
Взаимодествие пользователя с системой происходит через REST API, отправляя запросы на сервер и получая ответы в формате JSON.


Ролевая модель (роли в системе)
1. Неавторизованный - Просмотр публичных данных, регистрация, вход
2. USER	- Полный CRUD своих объявлений, комментарии, просмотр контактов
3. ADMIN - Управление пользователями, модерация всех объявлений


Основные бизнес-процессы
1. Неавторизованный пользователь
   - Регистрация / Логин
   - Просмотр всех объявлений (только публичные поля)
   - Фильтрация объявлений (по категории, цене, адресу)
   - Пагинация объявлений

2. Авторизованный пользователь
   - Доступен весь функционал CRUD объявлений (создание, просмотр, редактирование, удаление своих)
   - Просмотр контактных данных других пользователей
   - Комментирование объявлений
   - Просмотр приватных полей объявлений (телефон, имя пользователя определенного обьявления)

3. Администратор
   - Блокировка/разблокировка пользователей
   - Назначение ролей
   - Удаление любых объявлений
   - Массовое удаление всех объявлений  


Техническая реализация
1. Безопасность
   - JWT (JSON Web Tokens) для stateless-аутентификации 
   - Разделение доступа через роли USER и ADMIN 
   - Разграничение видимости данных (публичные и приватные поля)
   - Поддержка Basic Auth 

2. Работа с данными
   - PostgreSQL для хранения пользователей, объявлений, комментариев 
   - JPA/Hibernate для объектно-реляционного отображения 
   - Caffeine Cache для кэширования часто запрашиваемых данных 
   - Валидация входных данных

3. API и документация
   - RESTful API с четким разделением эндпоинтов 
   - Swagger/OpenAPI для автоматической документации 
   - DTO (Data Transfer Objects) для контроля возвращаемых данных

4. Архитектура
   - Многомодульный проект на Gradle (разделение на модули: domain — сущности, security — безопасность, 
app — здесь находятся основные настойки приложения и его запуск)
   - Многоуровневая архитектура: разделение на слои контроллеров, сервисов и репозиториев
